## 日本語

### このリポジトリについて

[`CLAUDE.md`](./CLAUDE.md) が本体で、全プロジェクトに適用される共通指示をまとめたもの。出力言語、コミットやリリースの作法、ドキュメントの書き方などを定めている。参考や流用の用途で公開している。

公開リポジトリのため、`CLAUDE.md` には個人のパス構成や名前など公開に適さない情報を新たに加えないようにしている（同趣旨の注意書きを `CLAUDE.md` 冒頭にも記載）。

### CLAUDE.md の内容

各節の概要は、`CLAUDE.md` の更新時にローカル LLM で変更節だけ要約して自動生成している。

<!-- AUTO:sections:ja -->
| 節 | 概要 |
| --- | --- |
| Language | 言語は日本語で出力。 |
| Post-coding completion output | 次へ進む / next / continue は、次のタスクや未来の機能を示す。 |
| Plan execution (after plan approval) | 作業を実行する際、サブアジェント駆動型（タスク毎に新しいサブアジェントを起動し、タスク間で二段階レビューを行う）を自動的に選ぶ。 |
| Git — author, commits, frequency | コミットタイトルは英語で表示し、本文は日本語または英語で、理由を含む。 |
| Git — safety (always-on) | デストラティブな操作は常に承認が必要。 |
| Conditional playbooks (load on demand) | 条件 playbooks は、ロードオンデマンドで、特定の状況に応じて使われる知識を含む。 |
<!-- /AUTO:sections:ja -->

全文は [`CLAUDE.md`](./CLAUDE.md)、変更履歴は [`CHANGELOG.md`](./CHANGELOG.md) を参照。

### About this repository

The core file is [`CLAUDE.md`](./CLAUDE.md): the shared Claude Code instructions applied to all of my projects, covering output language, commit and release conventions, documentation style, and more. It is published for reference and reuse.

Because the repository is public, `CLAUDE.md` is kept free of personal paths, names, or other details that should not be published; the same note appears at the top of `CLAUDE.md`.

### What CLAUDE.md contains

Each section summary is auto-generated by a local LLM, re-summarizing only the changed sections whenever `CLAUDE.md` is updated.

<!-- AUTO:sections:en -->
| Section | Summary |
| --- | --- |
| Language | Output must be in Japanese, and code, commands, filenames, and logs should be preserved as-is. |
| Post-coding completion output | The structure and requirements for documenting the results of a coding task, including summary, changes, verification, how to run, notes, next task, and commit proposal. |
| Plan execution (after plan approval) | Choose subagent-driven execution as the default for plan approval. |
| Git — author, commits, frequency | Git commits should follow a structured format with a clear title, use the user's own configuration, include a co-authored trailer, and maintain proper documentation practices. |
| Git — safety (always-on) | Destructive operations require approval, and remote deletions cannot be undone. |
| Conditional playbooks (load on demand) | Conditional playbooks enable dynamic version control, documentation, and style guidelines for localized development. |
<!-- /AUTO:sections:en -->

See [`CLAUDE.md`](./CLAUDE.md) for the full text and [`CHANGELOG.md`](./CHANGELOG.md) for history.
